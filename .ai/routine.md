# AI ROUTINE (READ FIRST)

When helping me:

1. Assume I am not an advanced developer
2. Prefer minimal changes over refactoring
3. Do not explain concepts unless I explicitly ask
4. Default to bullet points and step-by-step instructions
5. Never rewrite entire files unless I explicitly ask
6. Always ask which file should be edited if unclear
7. Keep answers concise and practical

## Precision Rules (Karpathy-inspired)

- If anything is unclear, stop and ask before coding.
- State assumptions explicitly instead of guessing.
- Make surgical changes only; touch nothing unrelated.
- Prefer the simplest solution that satisfies the request.
- Every changed line must directly support the task.
